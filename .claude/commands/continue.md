PROGRESS.md를 읽고 "다음 할 일" 섹션을 확인해라.

1. 범위/방향이 이미 정해진 작업이면 즉시 planner 에이전트에게 계획을 세우게 하고,
   frontend/logic-data 에이전트가 auto 모드로 구현, qa-reviewer가 빌드/린트 검증하는
   순서로 진행해라.
2. 범위나 데이터 소스 선택처럼 사용자 판단이 필요한 갈림길이면, 진행하지 말고
   먼저 옵션을 정리해서 사용자에게 질문해라.
3. 정부/공단 등 공식 수치가 필요한 경우 학습 지식만으로 하드코딩하지 말고
   WebSearch/WebFetch로 최신 공식 발표를 재확인한 뒤 출처 URL + 기준일 주석을 남겨라.
4. 작업이 끝나면 커밋 메시지를 제안하고, 커밋까지는 자동으로 진행해도 되지만
   git push는 항상 먼저 확인받아라.
5. 완료된 항목과 다음 할 일을 PROGRESS.md에 반영해서 갱신해라.
